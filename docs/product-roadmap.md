## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can run the app and see something functional — even if it's just a shell.
2. **Infrastructure before features.** Phase 0 sets up everything correctly so all future feature work is fast and clean. Don't skip it.
3. **Magic moment first.** The parent sees their first extracted homework card by end of Phase 2. Everything else builds on top of that.
4. **AI accuracy is a first-class concern.** Every extraction task includes a verification step against real school PDFs. Do not mark extraction tasks complete until tested against actual PDF samples.
5. **Test as you go.** Each task ends with a verification step. Don't batch verification — catch problems immediately before they compound.
6. **Review before proceeding.** After each phase, push a PR to GitHub and let a review agent ([CodeRabbit](https://coderabbit.ai) recommended — free for public repos) review it before starting the next phase. This catches issues early when they're cheap to fix.
7. **Parse anything.** ULTISchoolPulse's value is its flexibility across school formats. When implementing the AI extraction, prioritize handling varied PDF structures over handling the happy path.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a phase:** Give your coding agent the phase prompt at the beginning of each phase. The prompt tells it exactly which documents to read and what to build.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of the PRD and vision doc needed for that phase. The agent should read only those sections, not the entire documents.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, marks it complete, and moves to the next.
4. **One session = one phase (ideally):** Try to complete a full phase in a single coding session for best continuity. If you need to stop mid-phase, the agent can resume from the first unchecked task.
5. **Push a PR for review:** When a phase is complete, push the branch and open a PR. Use [CodeRabbit](https://coderabbit.ai) (free for public repos) to auto-review. Don't start the next phase until the PR is reviewed and merged.
6. **Need more context mid-task?** If a task references a section not in the phase's Reference sections, the agent should read just that section on demand rather than loading the whole document.

### Session Tips

- **Don't read everything upfront.** The PRD and vision doc are large. Each phase's Reference sections tell the agent exactly what to read.
- **Don't skip tasks.** Tasks are ordered intentionally — skipping creates dependency problems.
- **Verify after each phase.** Run the full app after each phase, not just the components you added. Integration bugs surface here.
- **Review before moving on.** A PR review catches issues when they're cheap to fix — before the next phase builds on top of them.
- **Update the status header.** After completing a phase, update `**Status:** X/65 tasks complete` and `**Current Phase:** Phase N`.
- **AI accuracy is your job too.** The extraction quality depends on real school PDFs. After Phase 1, test with your actual school's PDFs and tune the prompt before declaring Phase 1 complete.
